# Good Practices for Software Development Assignment


1. Which point(s) were new to you?  

The points that were new to me were the importance of over-communication in software development, joining meetings 5–10 minutes early to build better relationships with teammates, and combining all questions into one organized message instead of sending many separate texts. I also learned that programming mainly depends on sustained attention and deep work. Using screenshots, Loom videos, and GitHub Gists while asking technical questions was also a new and useful idea for me.


2. Which area do you think you need to improve on? What are your ideas to make progress in that area?  

I think I need to improve my focus management and communication skills. Sometimes distractions reduce my productivity, and I also need to communicate updates more clearly during teamwork. To improve, I will reduce social media usage during work hours, practice deep work with fixed study sessions, take proper notes during discussions, and communicate project updates regularly. I will also improve the way I ask technical questions by explaining problems clearly and mentioning the solutions I already tried.
